# WARDEN UI/UX & Design Language Document

**Product Name:** WARDEN (Wireless Advanced Response & Deployment Network)
**Document Version:** 1.0
**Date:** June 2026

---

## 1. Introduction
This document defines the visual aesthetic, user experience (UX) flow, and core design principles for the WARDEN ecosystem. Given the high-stakes, emergency-response nature of the application, the design prioritizes **situational awareness, cognitive ease, and high-contrast data visualization**.

## 2. Design Language & Philosophy
The overriding design language for WARDEN is **Tactical Modernism**. It leans heavily on clean, data-dense interfaces typically found in aerospace or military Heads-Up Displays (HUDs), combined with modern web affordances.

### 2.1 Core Principles
*   **Dark Mode First:** Reduces eye strain in low-light tactical environments and allows critical alert colors (Reds, Oranges) to pop immediately.
*   **Glanceability:** The most critical information (hazards, safe paths, casualty counts) must be understandable in under 2 seconds.
*   **Spatial Awareness:** Heavy emphasis on 2D/3D map integration and physical context over text-heavy tables.
*   **Micro-Animations:** Used strictly for drawing attention to state changes (e.g., pulsing red for new hazards, smooth path drawing for evacuation routes) rather than purely decorative purposes.

## 3. Global Color Palette
The color system uses deep neutral backgrounds accented by semantic colors indicating status.

### 3.1 Background & Surface Colors
*   **Base Background:** `#0B0F19` (Deep Tactical Navy/Black)
*   **Surface / Panel Backgrounds:** `#161D2C` (Dark Grey-Blue) with subtle glassmorphism (slight transparency and background blur) for overlay panels.
*   **Borders & Dividers:** `#2A3546`

### 3.2 Typography & Text
*   **Primary Text:** `#FFFFFF` (Pure White)
*   **Secondary Text:** `#94A3B8` (Slate Grey for less critical metadata)
*   **Disabled Text:** `#475569`

### 3.3 Semantic & Brand Accents
*   **Primary Brand (Teal/Command):** `#009688` — Used for primary actions, secure states, and command UI elements.
*   **Deep Tactical Accent:** `#005762` — Used for active states on the Desktop C&C applications.
*   **Safe Zone / Android Primary:** `#3DDC84` — Denotes safe evacuation routes, cleared areas, and online status.
*   **Hazard / Alert (Critical):** `#EF4444` — Reserved strictly for fire epicenters, critical injuries, and SOS triggers.
*   **Warning (Elevated):** `#F59E0B` — Smoke spread, high-density bottlenecks, or AI warnings.
*   **AI/Intelligence:** `#4285F4` (Google Blue) — Used to highlight insights, routes, or recommendations generated by the Gemini/Gemma engines.

## 4. Frontend Design by Application

### 4.1 Staff Desktop App (Command & Control)
*   **Layout:** A three-pane "Dashboard" layout.
    *   *Left Sidebar:* Global metrics, active alert stream, and AI tactical summaries.
    *   *Center Canvas:* The interactive 3D/2D digital twin of the building. This takes up 60% of the screen.
    *   *Right Sidebar:* The Unified ICS Task Board (drag-and-drop kanban style for assigning responder units).
*   **Key UX Flow:** Upon receiving an alert, the map auto-focuses on the hazard zone, the AI pane expands with tactical recommendations, and the user is prompted to broadcast an evacuation path with a single primary button.

### 4.2 Responder Desktop App (Field Units)
*   **Layout:** A streamlined, rugged interface optimized for quick taps on touch-enabled toughbooks or tablets.
*   **Key UX Flow:** Focuses purely on assigned ICS tasks and localized map views. The interface suppresses building-wide data to prevent cognitive overload, showing only the responder's current floor, hazard proximity, and their specific objective (e.g., "Clear Sector B").

### 4.3 Warden Android App (Occupants & Guests)
*   **Layout:** Clean, high-contrast, mobile-first design leveraging Material Design 3 principles adapted for emergencies.
*   **Key UX Flow:**
    *   *Default State:* Simple profile view, building map, and a large, accessible SOS button.
    *   *Active Emergency State:* The UI strips away all non-essential elements. The screen is dominated by a high-contrast directional arrow (indoor navigation) guiding the user along the AI-generated safe route, accompanied by an AI chat lifeline to Command.

## 5. Typography
*   **Font Family:** `Inter` or `Roboto` (sans-serif) for high legibility at all sizes.
*   **Monospace Font:** `Fira Code` or `JetBrains Mono` utilized specifically for raw data readouts, server logs, or AI token streams.
